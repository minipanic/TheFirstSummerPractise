# Отчет по летней производственной практике

Репозиторий содержит комплекс выполненных практических работ по проектированию реляционных баз данных (СУБД SQL Server), работе с HTTP-запросами, а также разработке консольных и веб-приложений на платформе .NET (C#).

## Структура репозитория

* **`Лабораторная работа 1, 2: типы данных`** (Реализовано в папках `DataTypes1`, `DataTypes1_`, `DataTypes2`)
  * Практические задания по изучению встроенных типов данных C#, механизмов приведения типов и работы с ветвлениями.
* **`Лабораторная работа 3, 4: Основы ООП и Коллекции`** (Реализовано в папках `Rectangle`, `Collection`)
  * Модули, демонстрирующие принципы объектно-ориентированного программирования (ООП), инкапсуляцию логики геометрических фигур и работу со стандартными коллекциями .NET.
* **`Лабораторная работа 5: Онлайн-магазин игр`** (Реализовано в папке **`DB`**)
  * Консольное CRUD-приложение для управления онлайн-магазином компьютерных игр.
  * Демонстрирует параллельное использование двух подходов доступа к данным: **ADO.NET** (прямые SQL-запросы) и **Entity Framework Core** (ORM).
  * Конфигурация строки подключения к СУБД вынесена в `appsettings.json`.
* **`Лабораторная работа 8: Веб-Приложение для учета личных финансов и бюджетирования`** (Реализовано в папке **`TradeApp`**)
  * Интерактивное веб-приложение для учета личных финансов и бюджетирования на базе **Blazor Web App** (.NET).
  * Реализована строгая бизнес-логика: проверка активности статей расходов при транзакциях и техническое ограничение на суточные траты (не более 1 000 000 рублей в день).
  * Внедрена цветовая индикация (зеленый/желтый/красный стикеры) для визуализации экономности выбранного дня.

## Техническая защита выполненных работ

### 1. База данных «Онлайн-магазин игр» (Связи и Индексы)
База данных спроектирована в среде SQL Server Management Studio (SSMS) и состоит из 5 связанных таблиц: `Games`, `Developers`, `Users`, `Orders`, `OrderItems`.
* **Реализованные связи:** `1:N` (один разработчик — много игр; один пользователь — много заказов) и `N:N` (игры и заказы через связующую таблицу позиций `OrderItems`).
* **Уникальный индекс:** Создан уникальный индекс `UX_Users_Email` на поле `Email` в таблице `Users`.
  * *Обоснование уместности:* Email выступает уникальным логином для авторизации. Индекс предотвращает появление дубликатов на уровне СУБД и ускоряет поиск учетной записи по электронной почте.
  * *Минусы решения:* Любые операции записи (`INSERT`, `UPDATE` поля Email) требуют дополнительных вычислительных ресурсов на перестройку b-дерева индекса. Также индекс занимает физическое дисковое пространство.

### 2. Тестирование HTTP-запросов и API (Выполнено в Postman)
В рамках изучения сетевых протоколов и интеграции со сторонними сервисами через **Postman** были успешно выполнены следующие сценарии:
* **GET-запрос к ЦБ РФ:** Получение актуальной структуры курсов валют на заданную дату в формате **XML**.
* **POST-запрос к Dummy API:** Создание новой учетной записи сотрудника с отправкой валидного **JSON**-тела и получением ответа от сервера с присвоенным ID.
* **GET-запрос к Dummy API:** Чтение полного массива пользователей (employees) в формате **JSON**.
